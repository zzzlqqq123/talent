# 个人天赋测试系统 - 产品需求文档（PRD）

## 1. 项目概述

### 1.1 项目背景
开发一款基于心理学和行为学的个人天赋测试网页应用，帮助用户全面了解自己的天赋优势，提供科学的能力评估和可视化报告。

### 1.2 项目目标
- 提供科学、专业的天赋能力评估
- 通过数据可视化直观展示用户天赋分布
- 记录用户成长轨迹，展示能力提升趋势
- 提供个性化的发展建议

### 1.3 目标用户
- 学生群体：了解自身优势，规划发展方向
- 职场人士：发现潜在能力，优化职业选择
- 心理爱好者：自我探索和认知提升

---

## 2. 功能需求

### 2.1 核心功能

#### 2.1.1 用户系统
**功能描述**：提供完整的用户账户管理功能

**详细需求**：
- 用户注册
  - 支持邮箱/手机号注册
  - 密码加密存储
  - 基础信息填写（昵称、年龄、性别等）
- 用户登录
  - 邮箱/手机号 + 密码登录
  - 记住登录状态（JWT Token）
  - 密码找回功能
- 个人中心
  - 查看和编辑个人资料
  - 修改密码
  - 查看测试历史记录
  - 管理保存的测试报告

**优先级**：P0（必须实现）

---

#### 2.1.2 天赋测试系统
**功能描述**：基于心理学和行为学的多维度测试问卷

**测试维度**：

1. **认知能力（Cognitive Ability）**
   - 逻辑思维：推理能力、问题分析能力
   - 记忆力：短期记忆、长期记忆
   - 注意力：专注度、持续性
   - 学习能力：新知识吸收速度

   *测试题数：20题*
   *评分方式：5点量表（1-5分）*

2. **创造力（Creativity）**
   - 想象力：形象思维、联想能力
   - 创新思维：突破常规、独特见解
   - 艺术感知：审美能力、艺术鉴赏

   *测试题数：15题*
   *评分方式：5点量表（1-5分）*

3. **情感智能（Emotional Intelligence）**
   - 同理心：理解他人情感
   - 情绪管理：自我情绪调控
   - 人际交往：沟通能力、团队协作

   *测试题数：15题*
   *评分方式：5点量表（1-5分）*

4. **实践能力（Practical Ability）**
   - 执行力：目标达成、行动力
   - 领导力：组织协调、决策能力
   - 适应力：环境适应、抗压能力

   *测试题数：15题*
   *评分方式：5点量表（1-5分）*

**问卷设计原则**：
- 总题数：65题
- 答题时间：约15-20分钟
- 题型：单选题、情景选择题
- 选项：李克特5点量表（完全不符合→完全符合）
- 逆向题设计：防止答题惯性，占总题数20%

**功能需求**：
- 问卷分页展示（每页10题，共7页）
- 答题进度条显示
- 支持暂存和继续答题
- 答题时间记录
- 答题有效性验证（防作弊：防快速点击）

**优先级**：P0（必须实现）

---

#### 2.1.3 评分与分析系统
**功能描述**：根据用户答题结果计算各项得分并生成分析

**评分算法**：
```
单项维度总分 = （正向题总分 + 逆向题转换分） × 10
维度得分 = 单项维度总分 / 题目数量
最终得分 = 维度得分 × 难度系数
```

**天赋等级划分**：
- 优秀（85-100分）：天赋突出
- 良好（70-84分）：能力较强
- 中等（50-69分）：能力正常
- 待提升（30-49分）：有发展空间
- 较弱（0-29分）：需重点提升

**分析内容**：
- 四大维度得分
- 各维度子项得分
- 天赋类型判定
- 优势能力识别（Top 3）
- 待提升能力识别（Bottom 2）
- 个人天赋关键词

**优先级**：P0（必须实现）

---

#### 2.1.4 报告生成系统
**功能描述**：生成可视化的测试报告

**报告内容结构**：

1. **基本信息**
   - 测试日期
   - 测试用时
   - 测试编号

2. **总体评估**
   - 综合得分
   - 天赋类型标签
   - 一句话评价

3. **数据可视化**
   - **雷达图**：四大维度能力分布
   - **柱状图**：12个子项得分对比
   - **词云图**：天赋关键词展示

4. **详细分析**
   - 各维度详细评语
   - 优势能力说明
   - 发展建议

5. **对比分析**
   - **趋势图**：历史测试得分变化
   - 同龄人对比（可选）

6. **操作选项**
   - 保存报告
   - 分享报告（生成分享链接）
   - 下载PDF报告
   - 重新测试

**优先级**：P0（必须实现）

---

#### 2.1.5 历史记录与对比
**功能描述**：记录用户所有测试历史，支持对比分析

**功能需求**：
- 测试历史列表
  - 显示：测试时间、总分、主要天赋标签
  - 支持按时间排序
  - 支持筛选查看
- 历史详情查看
  - 查看完整报告
  - 删除历史记录
- 对比功能
  - 选择2-5次测试进行对比
  - 生成对比趋势图
  - 能力变化分析

**优先级**：P1（重要功能）

## 3. 非功能需求

### 3.1 性能要求
- 页面加载时间 < 2秒
- 报告生成时间 < 3秒
- 支持500并发用户
- 数据库查询响应 < 500ms

### 3.2 安全要求
- 密码使用bcrypt加密
- API使用JWT认证
- 防SQL注入
- 防XSS攻击
- HTTPS传输

### 3.3 兼容性要求
- 支持浏览器：Chrome、Firefox、Safari、Edge（最新2个版本）
- 移动端适配：响应式设计
- 屏幕分辨率：1366×768 及以上

### 3.4 可用性要求
- 系统可用性 > 99%
- 数据备份：每日自动备份
- 错误日志记录

---

## 4. 数据需求

### 4.1 核心数据实体
- 用户信息
- 测试题目
- 用户答题记录
- 测试结果报告
- 历史记录

### 4.2 数据保留策略
- 用户数据：永久保存（除非用户注销）
- 测试记录：永久保存
